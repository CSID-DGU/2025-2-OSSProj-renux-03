# Memory

## 진행 상황 요약
- `index.html`에 부트스트랩 모달을 추가하고 `새 채팅` 버튼을 단일 트리거로 재구성해, 누구나 버튼을 누르면 학과 선택/제목 입력 모달이 뜨도록 UI를 정비함.
- `css/index.css`에서 새 채팅 버튼 스타일과 비활성화 상태를 정리해 챗지피티처럼 깔끔한 사이드바 UX를 구성함.
- `js/index.js`를 전면 개편하여 `/req/orgs`로 학과를 불러와 `<select>` 옵션을 채우고, 모달에서 `{ orgId, title }`로 `/chat/new`에 POST 한 뒤 신규 채팅 ID로 라우팅하는 흐름을 구현함.
- 학과 목록이 없는 경우에도 모달을 열어 UX 흐름을 확인할 수 있고, 서버 응답이 준비되면 자동으로 생성 버튼이 활성화되도록 대기 상태(알림 포함) 로직을 추가함.

## 앞으로의 계획
1. 백엔드에서 `/chat/new`가 `{ orgId, title }`을 받아 채팅을 생성하고 `{ id }`를 반환하도록 연동 검증 및 오류 처리를 강화한다.
2. 학과 목록 API `/req/orgs` 응답 포맷을 프런트와 맞춰 확정하고, 실패 시 대체 메시지/로그를 정리한다.
3. 새 채팅으로 이동한 뒤 실시간 메시징(소켓/폴링) 초기화 흐름을 정의하고 필요한 추가 UI 반응(로딩, 에러 핸들링 등)을 보완한다.
